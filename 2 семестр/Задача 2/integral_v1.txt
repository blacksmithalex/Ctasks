Для выбранного метода численного интегрирования:
1) Пишем функцию для численного подсчета интеграла.
Аргументы функции:
а) концы отрезка,
б) количество разбиений
в) функция от которой брать интеграл.
intergral(double a, double b, int N, double (*f)(double) )

2) Проводим исследование.
Для функций из списка:
а) константа,
б) линейная,
в) парабола,
г) sin
исследуем, что будет с численным значением I_n интеграла I с изменением N.
Замечу,
что для заданных функций мы можем посчитать первообразную, а значит и истинный ответ I.
Прототип функций аналогичен тому, что был на численный поиск корня.

Для некоторых функций уже при N=1 численный интеграл I_1 будет давать идеальный ответ (в таком случае он, I_n, должен быть идеальным и с ростом N).
Для других функций численное интегрирование I_n даст погрешность.
В последнем случае нужно понять:
а) как меняется ошибка с ростом N (например, в два раза): I_n - I.
б) как меняется разница между численно посчитанными интегралом с ростом N: I_n - I_2n.
Для данных величин конечно записывается абсолютная величина.
Погрешности по аналогии с поиском корня выводим через %e.
